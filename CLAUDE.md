# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a single-cell RNA-seq analysis pipeline built on Seurat V5. It processes 10X Genomics Chromium data (standard, multiome ATAC+GEX, and xenograft) through a staged workflow: QC/filtering → normalization/integration → dimensionality reduction/clustering → cell-type annotation.

Scripts are run from an **analysis working directory** (not from this `scRNA/` repo directory). The repo is pointed to via `SDIR` env variable or `getSDIR()` from `~/.Rprofile`.

## Pipeline Stages

| Stage | Script | Description |
|-------|--------|-------------|
| 1 | `doSeuratV5_01.R` | Load 10X data, QC plots, cell cycle scoring |
| 1-filter | `doSeuratV5_01_Filter.R` | Interactive filter testing (reads preFilter cache) |
| 2a | `doSeuratV5_02a_MergeOnly.R` | SCTransform + simple merge |
| 2a-alt | `doSeuratV5_02a_IntegrateData.R` | SCTransform + Seurat integration |
| 2b | `doSeuratV5_02b.R` | PCA, UMAP, clustering, module scores |
| 3+ | `doSeuratV5_03_*.R` | Post-clustering: cell type annotation, feature plots, heatmaps, etc. |

## Running the Pipeline

**Automated (recommended):** Use `CMDS.p12` bash script from the analysis working directory:

```bash
export SDIR=/path/to/scRNA

# Standard run (merge mode, auto-detect cellranger dirs)
bash $SDIR/CMDS.p12

# With explicit config (multiome/multiplex samples)
bash $SDIR/CMDS.p12 --config config.yaml

# Integration mode instead of merge
bash $SDIR/CMDS.p12 --integrate

# Stop after filtering (stages 1+2a only)
bash $SDIR/CMDS.p12 --filter

# With gene/cell filters
bash $SDIR/CMDS.p12 --genefilter genes.txt --cellfilter cells.csv
```

**Manual (no-merge example):**
```bash
export SDIR=scRNA
python3 $SDIR/extractProjectIDFromPath.py $PWD | sed 's/Proj_/p/' >PROJNAME
Rscript --no-save $SDIR/doSeuratV5_01.R ../cellRanger/s_*
Rscript --no-save $SDIR/doSeuratV5_02a_MergeOnly.R CC_REGRESS=FALSE pass_01_PARAMS.yaml
Rscript --no-save $SDIR/doSeuratV5_02b.R pass_02_PARAMS.yaml
```

## Configuration

**`pass_00_PARAMS.yaml`** — QC filter overrides (in analysis working dir). Defaults:
```yaml
algoParams:
  MIN_FEATURE_RNA: 1500
  MIN_NCOUNT_RNA: 5000
  PCT_MITO: 10
  METHOD: _auto   # or MAD3
  XENO_PCTCUT: 80
```
Run stage 1 first; it writes `pass_00_PARAMS.yaml.mad3` with MAD3-computed cutoffs as a starting point.

**`config.yaml`** — Required for multiome/multiplex runs (explicit sample paths):
```yaml
genome: mm10   # or hg38, xenograft
multi:
  gex: "Gene Expression"
inputs:
- dir: /path/to/sample_filtered_feature_bc_matrix
  sid: SAMPLE-ID
```
Use `getCRConfigTmpl.sh <CELL_RANGER_DIR>` to generate a template.

**`PROJNAME`** file — single-line project name (auto-generated by `CMDS.p12`).

## Inter-stage Parameter Files

Each stage writes a YAML params file consumed by the next:
- Stage 1 → `pass_01_PARAMS.yaml`
- Stage 2a → `pass_02_PARAMS.yaml`
- Stage 2b → `pass_02b_PARAMS.yaml` (consumed by stage 3 scripts)

## Shared Utility Scripts

- `seuratTools.R` — Seurat helpers: `read10XDataFolderAsSeuratObj`, `read_10X_multi_as_SeurateObject`, `scoreCellCycle`, `plotCellCycle`, `preProcessSO`, `get_plot_filename`, `makeAutoIncrementor`
- `plotTools.R` — ggplot2 helpers
- `qcAndFilter.R` — `qcSamples()` QC metrics and filtering

These are sourced at runtime via `source(file.path(SDIR, "seuratTools.R"))` etc.

## Output Structure

Results go to `results/stage{N}/` in the analysis working directory:
- `stage1/` — QC, cell cycle, xeno stats PDFs
- `stage2/` — PCA, UMAP, cluster charts, sample bias plots
- `stage3/` — Cluster markers, dot plots, heatmaps, UMAPs by resolution

Plot filenames follow: `{PROJNAME}_plt_{NN}_{description}.pdf`

## Data

- `data/genomes.yaml` — genome definitions (hg38, mm10, xenograft)
- `data/cc.genes.mouse.v2.rda` — mouse cell cycle genes
- `data/genes_Ribosomal_Human`, `data/genes_Ribosomal_Mouse` — ribosomal gene lists
- `data/celldex_ImmGenData_Annote.csv.gz` — cached ImmGen reference annotations

## Key Design Patterns

- Scripts use `KEY=VALUE` command-line argument parsing (not standard flags)
- `SDIR` env variable points to this repo; scripts `source()` utilities from it
- Each stage saves an RDS cache and PARAMS yaml for the next stage
- `digest::digest()` is used to fingerprint inputs for cache validation
- R >= 4.0 required; conda env at `Work/CONDA/VirtualEnvs/scRNA_V4`
